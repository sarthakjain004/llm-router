# llm-router

One OpenAI-compatible endpoint for your LangGraph agents, with automatic
failover across **free-tier** LLM APIs — so inference stays at $0.

Built on [LiteLLM](https://docs.litellm.ai). Runs in Docker on ARM64 (Apple
Silicon for local dev, Oracle Cloud Always Free for prod).

```
                        ┌──────────────────────────── llm-router (:4000/v1) ───────────────────────────┐
  LangGraph agent  ──▶  │  semantic alias        fallback chain (router_settings.fallbacks)             │
  base_url=:4000/v1     │  ─────────────         ────────────────────────────────────────────          │
  model="agent-default" │  agent-default  ──▶  Gemini ▸ Groq ▸ Cerebras ▸ OpenRouter:free ▸ NVIDIA NIM  │
  api_key=MASTER_KEY    │  vision         ──▶  Groq Scout ▸ NIM (vision-capable)                        │
                        │  long-context   ──▶  Gemini Flash-Lite ▸ Groq ▸ NIM                           │
                        └──────────────────────────────────────────────────────────────────────────────┘
                             per-provider 429/5xx → cooldown, then next tier. All OpenAI-compatible.
```

- **Fully OpenAI-compatible**: `/v1/chat/completions` (+ streaming SSE + tool/function calls), `/v1/models`.
- **Never pay**: only free tiers, one legit key per provider.
- **Self-healing**: a 429 or 5xx benches a provider for a cooldown and the next tier serves the request.
- **Deploy-ready**: `docker compose up`, an ARM64 runbook for Oracle Cloud, and a smoke + failover test.

## 1. The alias contract (what your agents call)

Your agents should only ever reference these three **semantic aliases**. Their
meaning is stable even as providers/models churn underneath:

| Alias           | Use it for                          | Primary → fallbacks |
| --------------- | ----------------------------------- | ------------------- |
| `agent-default` | general agent work                  | Gemini 2.5 Flash → Gemini Flash-Lite → Groq Llama-70B → Cerebras gpt-oss → OpenRouter `:free` → NVIDIA NIM |
| `vision`        | image inputs                        | Gemini 2.5 Flash → Groq Llama-4-Scout → NIM Scout |
| `long-context`  | very large prompts                  | Gemini 2.5 Flash → Gemini Flash-Lite → Groq Scout → NIM |

There are also **provider-pinned aliases** (`gemini-flash`, `groq-llama70b`,
`groq-fast`, `cerebras-gptoss`, `openrouter-free`, `nim-llama`, …) for debugging
or deliberately pinning one provider. By design these **do not fail over** — a
pinned alias that fails, fails loudly. That's the point of pinning.

### Adding a `smart` / `fast` tier later (no agent-code change)

The semantic layer is separate from the provider layer, so a new tier is two
edits in `config.yaml`:

```yaml
# 1) a new model_list entry (the primary for the tier)
- model_name: fast
  litellm_params: { model: groq/llama-3.1-8b-instant, api_key: os.environ/GROQ_API_KEY, rpm: 30, tpm: 6000 }
  model_info: { supports_function_calling: true, max_input_tokens: 131072 }

# 2) its chain, in router_settings.fallbacks
  - fast: [groq-fast, cerebras-gptoss, gemini-flash-lite]
```

Agents opt in by setting `model="fast"`. Nothing else changes. See
[ADR-0002](docs/adr/0002-two-layer-alias-architecture.md).

## 2. Providers & free-tier limits

Numbers are annotations for the router's pre-call checks (verified July 2026 —
they drift; re-check with `python3 scripts/refresh_models.py --check`).

| Priority | Provider    | Good free model (VERIFY-AT-BUILD)         | Free limits (approx)              | Notes |
| -------- | ----------- | ----------------------------------------- | --------------------------------- | ----- |
| 1        | Google Gemini (AI Studio) | `gemini-2.5-flash`          | ~10 RPM / 250 RPD / 250K TPM      | Only free 1M-context + strong vision. **Free-tier prompts may be used for training.** |
| 2        | Groq        | `llama-3.3-70b-versatile`                 | 30 RPM / 1K RPD / **100K TPD**    | Fast. TPD is the real ceiling. Aggressive model deprecations. |
| 3        | Cerebras    | `gpt-oss-120b`                            | **5 RPM** / 30K TPM / 1M tok/day  | Very fast, very low RPM. |
| 4        | OpenRouter  | `openrouter-free` group (auto-refreshed)  | 20 RPM / **50 requests/day**      | `:free` lineup rotates → generated by the refresh script. Renews daily. |
| 5        | NVIDIA NIM  | `meta/llama-3.3-70b-instruct`             | 40 RPM, **finite ~1000 credits**  | **Last-resort reserve**: credits don't renew. Kept last on purpose. |
| (opt)    | Ollama      | `qwen3:4b` (local, CPU)                   | unlimited but slow                | Skipped by default; commented stubs in compose + config. |

Why OpenRouter (#4) sits before NIM (#5) even at 50/day: OpenRouter's quota
**renews daily** (unused quota evaporates), while NIM's ~1000 credits are
**finite and non-renewing**. Spend the renewable resource first, keep the
finite one as the emergency reserve. To swap them, swap the two names in the
`agent-default` chain in `config.yaml`. See
[ADR-0003](docs/adr/0003-provider-fallback-ordering.md).

## 3. Quickstart (local, macOS Apple Silicon)

Prereqs: Docker Desktop (or colima) with the compose plugin running.

```bash
cp .env.example .env
make genkey            # prints an sk-... master key; paste into .env
$EDITOR .env           # set LITELLM_MASTER_KEY + at least GEMINI_API_KEY and GROQ_API_KEY

python3 scripts/refresh_models.py --write   # populate the openrouter-free group (needs no key)

make up                # build (native arm64) + start
make smoke             # liveness, models, chat, streaming, tool call, mocked fallback
```

`make up` = `docker compose up -d --build`. The endpoint is
`http://127.0.0.1:4000/v1`; agents authenticate with `LITELLM_MASTER_KEY`.

A leave-a-line-blank rule: if you don't have a provider's key yet, leave it
empty in `.env`. Those deployments fail fast and the router skips them.

## 4. Point a LangGraph app at it

```python
from langchain_openai import ChatOpenAI

llm = ChatOpenAI(
    base_url="http://127.0.0.1:4000/v1",   # your proxy (Tailscale IP in prod)
    api_key="<LITELLM_MASTER_KEY>",
    model="agent-default",                 # or "vision" / "long-context"
    timeout=120,
    max_retries=0,                         # the proxy already retries + fails over
)
```

Use it anywhere LangGraph takes a model (`create_react_agent(llm, tools=...)`).
Streaming and streamed tool-calls pass straight through. A fuller example is in
[`examples/langgraph_client.py`](examples/langgraph_client.py).

## 5. Keeping the free-model list fresh

`:free` models appear and disappear. `scripts/refresh_models.py` regenerates the
`openrouter-free` load-balanced group in `config.yaml` (between `BEGIN/END
GENERATED` markers) and validates the hand-pinned model ids against each
provider's `/models` endpoint (warn-only — it never rewrites pinned entries).

```bash
python3 scripts/refresh_models.py --check           # dry run: diff + validation (exit 2 = drift)
python3 scripts/refresh_models.py --write           # apply
python3 scripts/refresh_models.py --write --restart # apply + restart the container if it changed
```

It's stdlib-only (no pip install) and byte-idempotent (a no-op if nothing
changed). Suggested cron on the Oracle box:

```cron
# daily drift check (logs a warning if a pinned model vanished upstream)
17 6 * * *  cd /opt/llm-router && /usr/bin/python3 scripts/refresh_models.py --check  >> /var/log/llm-router-refresh.log 2>&1
# weekly refresh + restart
47 6 * * 0  cd /opt/llm-router && /usr/bin/python3 scripts/refresh_models.py --write --restart >> /var/log/llm-router-refresh.log 2>&1
```

See [ADR-0006](docs/adr/0006-generated-openrouter-block.md).

## 6. Oracle Cloud Always Free deploy (Ubuntu ARM64)

The box: 2 vCPU / 12 GB / aarch64. The images are multi-arch, so no emulation.

```bash
# 1) Docker + compose plugin
sudo apt-get update && sudo apt-get install -y docker.io docker-compose-v2
sudo systemctl enable --now docker
sudo usermod -aG docker "$USER"      # log out/in for group to take effect

# 2) Private networking with Tailscale (recommended)
curl -fsSL https://tailscale.com/install.sh | sh
sudo tailscale up                    # note the 100.x.y.z tailnet IP it prints

# 3) The app
sudo mkdir -p /opt/llm-router && sudo chown "$USER" /opt/llm-router
git clone <your-repo-url> /opt/llm-router && cd /opt/llm-router
scp your-mac:/path/to/.env ./.env    # copy your .env over — never commit it
# set BIND_ADDR to the tailnet IP so 4000 is reachable ONLY over the tailnet:
sed -i 's/^BIND_ADDR=.*/BIND_ADDR=100.x.y.z/' .env

python3 scripts/refresh_models.py --write
docker compose up -d --build
HOST=http://100.x.y.z:4000 ./scripts/smoke.sh   # run from a device on the tailnet
```

**Firewall — read this.** Docker-published ports **bypass ufw**, so the
`BIND_ADDR` in `.env` is your real access control, not a firewall rule:

- Do **not** add an ingress rule for TCP 4000 in the Oracle *Security List / NSG*.
- Leave Oracle Ubuntu's default-REJECT iptables (`/etc/iptables/rules.v4`) alone.
- With `BIND_ADDR` = the tailnet IP, port 4000 is unreachable from the public
  internet; verify: from off-tailnet, `curl http://<public-ip>:4000/health/liveliness`
  must fail.
- No-Tailscale alternative: keep `BIND_ADDR=127.0.0.1` and tunnel:
  `ssh -L 4000:127.0.0.1:4000 ubuntu@<box>`.

`restart: unless-stopped` + `systemctl enable docker` means the container comes
back on its own after a crash or VM reboot — no systemd unit needed. Security
rationale in [ADR-0007](docs/adr/0007-security-master-key-private-network.md).

## 7. Optional toggles

**OpenTelemetry** (for your OpenLIT/OTel stack) — off by default. In `.env`:

```bash
OTEL_ENABLED=true
OTEL_EXPORTER=otlp_http
OTEL_EXPORTER_OTLP_ENDPOINT=http://your-openlit-host:4318
```

then `make restart`. The entrypoint uncomments `callbacks: ["otel"]` in the
config at boot; when `false`, no exporter runs and there's zero OTel log noise.
See [ADR-0004](docs/adr/0004-otel-toggle-sentinel-entrypoint.md).

**Postgres logging via Neon** — off by default. Uncomment `DATABASE_URL` in
`.env` (LiteLLM runs Prisma migrations automatically on boot) and `make restart`.
**Never set `DATABASE_URL=` to an empty string** — an empty value still triggers
Prisma and crash-loops the container; comment the whole line to disable. See
[ADR-0005](docs/adr/0005-postgres-neon-optional.md).

**Ollama local fallback** — skipped by default. Uncomment the `ollama` service
in `docker-compose.yml` and the `ollama-local` entry in `config.yaml`, then:

```bash
docker compose up -d ollama
docker exec ollama ollama pull qwen3:4b
# set OLLAMA_API_BASE=http://ollama:11434 in .env, append `ollama-local` to the
# agent-default fallback chain, then: make restart
```

## 8. Proving failover

**Mocked** (fast, no real failure) — the smoke test does this via the
`mock_testing_fallbacks: true` request flag.

**Real** — `scripts/failover_test.sh` recreates the proxy with a poisoned Gemini
key (`compose.failover.yml`), sends an `agent-default` request, and asserts
**Groq** served it (Gemini and Gemini Flash-Lite share the dead key, so the first
working fallback is `groq-llama70b`), then restores the healthy stack:

```bash
make failover
```

You can see which provider served any request three ways: the response body's
`model` field, the `x-litellm-*` response headers, and
`docker compose logs litellm | grep -iE 'fallback|cooldown'`.

## 9. Troubleshooting

| Symptom | Likely cause / fix |
| ------- | ------------------ |
| Every request 429s and cycles providers | You're hitting daily caps (RPD/TPD), not per-minute. Raise `cooldown_time` to `300` in `config.yaml`. |
| `openrouter-free` missing from `/v1/models` | Run `python3 scripts/refresh_models.py --write` (the group starts empty). |
| Groq requests suddenly fail | Groq deprecates models often. Run `--check`; replace the pinned id if warned. |
| Container crash-loops on boot | Usually `DATABASE_URL=` set to empty. Comment the line out entirely. |
| Which provider actually served? | Response `model` field, `x-litellm-*` headers, or the log grep above. |
| `/health` hangs / is slow | `/health` (authenticated) calls **every** model and spends NIM credits. Use `/health/liveliness` for probes. |
| Leaked master key | Rotate `LITELLM_MASTER_KEY` in `.env`, `make restart`, update agents. |
